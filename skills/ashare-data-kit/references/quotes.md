# Layer 1 行情层 — quotes.py 参考

实时行情 + K线 + 盘口。**这一层全部不封 IP**,可放心高频调用。

## 命令

```bash
uv run scripts/quotes.py kline 600519 --period day --count 60   # mootdx K线
uv run scripts/quotes.py realtime 600519 000858 sh000001        # 腾讯:价/PE/PB/市值/涨跌停
uv run scripts/quotes.py sina 600519 sh000001                   # 新浪:价+五档盘口(兜底)
uv run scripts/quotes.py kline-ma 600519                        # 百度:K线自带MA5/10/20
uv run scripts/quotes.py transaction 600519 --date 20260612     # mootdx 逐笔成交
# 任意命令加 --json 输出原始 JSON
```

`--period`: `day/week/month/1m/5m/15m/30m/60m`。

## 四个行情源怎么选

| 源 | 命令 | 给什么 | 不给什么 | 封IP |
|----|------|--------|----------|------|
| mootdx(通达信 TCP) | `kline` / `transaction` | K线 / 五档 / 逐笔成交 | PE/PB/市值/换手率 | 否 |
| 腾讯(HTTP GBK) | `realtime` | 价 + **PE/PB/市值/换手率/涨跌停/量比** | — | 否 |
| 新浪(HTTP GB18030) | `sina` | 价 + 五档盘口(量=股) | PE/PB/换手率/涨跌停 | 否 |
| 百度(HTTP) | `kline-ma` | K线**自带均线** | 实时盘口 | 否 |

**经验法则:** 要估值字段(PE/PB/市值)用腾讯 `realtime`;腾讯偶发抽风时用新浪 `sina` 兜底(两家都不封 IP);要均线用 `kline-ma`(优先百度,百度 403 时自动用腾讯/新浪/mootdx K线本地算 MA5/10/20);要 K线/逐笔/财务用 mootdx(需国内 IP,海外走代理)。

## 新浪 vs 腾讯(两个实测坑)

- **新浪必须带 `Referer: http://finance.sina.com.cn`**,否则 `403`;编码 `GB18030`,`,` 分隔,A 股约 32 字段 + 状态位,**不含 PE/PB/换手率**。
- 新浪成交量单位是**股**;腾讯字段 6 的成交量单位是**手**(×100 转股)。
- **GBK/GB18030 + 分隔符安全:** 用 `requests` + 先 `r.encoding="gbk"/"gb18030"` 解码再 split 是安全的——`~`/`,` 在解码后的字符串里只出现在真正的分隔位。只有直接处理**裸字节流**时才会被 0x7E 落在汉字低字节区切碎(嵌入式场景才需用 `~<code>~` 锚点跳过名字段)。本 skill 走 requests,naive split 安全。
- **港股/美股**(腾讯 `hk00700` / `usAAPL`——美股**不要**加 `.OQ`/`.N` 后缀,否则 `v_pv_none_match`;新浪美股要小写 `gb_aapl`)。本 skill 默认 A 股,如需港美股自行拼 symbol。

## 指数 / ETF(代码歧义)

指数代码与深市股票代码会撞车:`000001` 既是上证指数(sh)也是平安银行(sz)。`market_symbol()` **尊重显式前缀**:
- `sh000001` → 上证指数;`000001`(无前缀)→ 平安银行(sz)。
- 指数:`sh000001` 上证 / `sh000300` 沪深300 / `sz399006` 创业板指。ETF:`sh510050` 上证50ETF / `sh510300` 沪深300ETF。

## mootdx category(K线周期)

`4=日 5=周 6=月 7=1min 8=5min 9=15min 10=30min 11=60min`(`quotes.py kline --period` 已封装映射)。mootdx 走 TCP 7709,**需国内 IP**才稳定,海外环境走代理;腾讯/新浪/百度不受影响。

## kline-ma 兜底

百度股市通部分 IP 会返回 `ResultCode=403`。`kline-ma` 现在按 `百度 → 腾讯 K线 → 新浪 K线 → mootdx K线` 顺序兜底,统一输出 `{keys, rows, source}`。兜底源会在 `source` 标明,如 `tencent_fallback`。
